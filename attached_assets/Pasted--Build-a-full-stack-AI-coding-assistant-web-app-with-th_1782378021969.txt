 
Build a full-stack AI coding assistant web app with this exact spec:
 
STACK:
- Backend: Python FastAPI with uvicorn
- Frontend: Single HTML file (no React, no npm)
- AI: Google Gemini 1.5 Flash API (free)
 
LAYOUT:
- Full-viewport split: left 50% = code editor, right 50% = chat
- Header bar with app title and model name
- Dark theme throughout (#0f1117 background)
 
LEFT PANEL (code editor):
- CodeMirror 5 editor with Dracula theme and Python syntax highlighting
- Toolbar buttons: Explain | Debug | Write Tests | Optimize | Run ▶ | Clear
- Output box below the editor showing stdout/stderr after running code
 
RIGHT PANEL (chat):
- Scrollable message list with user and assistant bubbles
- Assistant messages render code blocks with syntax highlighting (highlight.js)
- Copy button on every code block
- Typing indicator (3 bouncing dots) while waiting for response
- Quick-action chips: "What does this do?" | "Security check" | "Add type hints" | "Make async" | "Clear chat"
- Text input + Send button (Enter to send, Shift+Enter for newline)
 
BACKEND ENDPOINTS:
POST /chat — body: { message, code, history[], action }
  - action can be: chat | explain | debug | test | optimize
  - Each action prepends a specific instruction to the prompt
  - Keep last 20 turns of history
  - System prompt makes it a strict coding assistant
 
POST /run — body: { code, language }
  - Runs Python code in subprocess with 10 second timeout
  - Returns { output, error }
 
GET /health — returns { status: "ok" }
 
GEMINI INTEGRATION:
- Use google-generativeai Python package
- Model: gemini-1.5-flash
- API key from environment variable GEMINI_API_KEY
- Use start_chat() with history for multi-turn conversations
 
ENVIRONMENT:
- Create a .env file with GEMINI_API_KEY=YOUR_KEY_HERE
- Use python-dotenv to load it
- Add .env to .gitignore
 
REQUIREMENTS (requirements.txt):
fastapi, uvicorn[standard], google-generativeai, python-dotenv, pydantic
 
START COMMAND: uvicorn main:app --host 0.0.0.0 --port 8000 --reload
